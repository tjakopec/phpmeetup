─────────────────────────────────────────────────────────────
0. OVERVIEW
─────────────────────────────────────────────────────────────
Build a stateless REST endpoint (POST /api/shipping/quotes) on
top of Symfony 7 + API Platform 3 that returns a shipping price
quote for parcels within Croatia. No database writes on quote;
reads only zone/tariff config. No carrier API integrations.

2. DATABASE / DOMAIN MODEL
─────────────────────────────────────────────────────────────
Two Doctrine entities (infrastructure/persistence only — not
used as DTOs).

2.1 Entity: PostalCode
    Table: postal_codes
    Fields:
      id         INT PK AUTO_INCREMENT
      code       VARCHAR(5) UNIQUE NOT NULL   — e.g. "10000"
      city       VARCHAR(100) NOT NULL
      zone_id    INT FK → shipping_zones.id

    Index: UNIQUE(code)

2.2 Entity: ShippingZone
    Table: shipping_zones
    Fields:
      id           INT PK AUTO_INCREMENT
      name         VARCHAR(50) NOT NULL       — "mainland", "island", "remote"
      zone_surcharge NUMERIC(8,2) NOT NULL    — EUR surcharge per quote
      description  TEXT NULL

2.3 Entity: WeightTariff
    Table: weight_tariffs
    Fields:
      id               INT PK AUTO_INCREMENT
      zone_id          INT FK → shipping_zones.id
      min_weight       NUMERIC(8,3) NOT NULL  — kg, inclusive lower bound
      max_weight       NUMERIC(8,3) NULL      — kg, NULL = no upper limit
      base_price       NUMERIC(8,2) NOT NULL  — EUR base for this bracket
      weight_unit_price NUMERIC(8,4) NOT NULL — EUR per kg above min_weight

    Rationale: tiered brackets, ordered by min_weight ASC.
    No pricePerKg flat model.

Migration: generate via doctrine:migrations:diff after entity creation.

6. DOMAIN SERVICE — CALCULATOR
─────────────────────────────────────────────────────────────
6.1 Interface: App\Service\ShippingPriceCalculatorInterface
    Method:
      calculate(ShippingQuoteInput $input): ShippingQuoteOutput

    This is the future seam for swapping in real carrier rates.

6.2 Implementation: App\Service\ShippingPriceCalculator

    Constructor injects:
      PostalCodeRepositoryInterface  $postalCodeRepo
      WeightTariffRepositoryInterface $tariffRepo
      ShippingConfiguration           $config

    calculate() algorithm:
      a) Resolve postal code → ShippingZone
         - Query PostalCode entity by code
         - If not found → throw UnresolvablePostalCodeException (→ 422)

      b) Validate hard limits BEFORE pricing
         - actualWeight > config.max_weight[serviceType] → throw OversizedParcelException (422)
         - max(L, W, H) > config.max_dimension[serviceType] → throw OversizedParcelException (422)

      c) Compute volumetric weight
         volumetric = (length * width * height) / volumetric_divisor
         chargeableWeight = max(actualWeight, volumetric)

      d) Find tariff bracket
         - Query WeightTariff for zone, ordered by min_weight ASC
         - Select first bracket where chargeableWeight >= min_weight
           AND (max_weight IS NULL OR chargeableWeight <= max_weight)
         - If no bracket → throw NoTariffFoundException (500 / config error)

      e) Compute base price from bracket
         base = bracket.base_price
               + (chargeableWeight - bracket.min_weight) * bracket.weight_unit_price

      f) Weight surcharge
         weightSurcharge = max(0, chargeableWeight - actualWeight)
                           * bracket.weight_unit_price
         (charge for the extra chargeable weight over actual)
         Note: if chargeableWeight == actualWeight → weightSurcharge = 0.00
               if chargeableWeight > actualWeight → weightSurcharge reflects
               volumetric premium — adjust for clarity:
         Simpler: weightSurcharge = (chargeableWeight > actualWeight)
                    ? (chargeableWeight - actualWeight) * bracket.weight_unit_price
                    : 0.00

      g) Dimensional surcharge
         dimensionalSurcharge = (chargeableWeight > actualWeight)
                    ? (chargeableWeight - actualWeight) * bracket.weight_unit_price
                    : 0.00
         (If volumetric > actual, dimensional surcharge equals the difference cost)
         Note: weightSurcharge and dimensionalSurcharge are mutually exclusive
         in this model — only one can be non-zero at a time. The breakdown
         surfaces them separately for clarity as requested.
         REVISED model:
           weightSurcharge     = 0.00 always (base already covers weight tiers)
           dimensionalSurcharge = (volumetric > actual)
                    ? (volumetric - actual) * bracket.weight_unit_price : 0.00

      h) Zone surcharge
         zoneSurcharge = zone.zone_surcharge

      i) Priority multiplier
         priorityMultiplier = (serviceType == 'priority')
                    ? config.priority_multiplier : 1.0

      j) Final price
         subtotal = base + dimensionalSurcharge + zoneSurcharge
         price    = round(subtotal * priorityMultiplier, 2)

      k) Estimated delivery days
         deliveryDays = (serviceType == 'priority')
                    ? config.priority_delivery_days_base
                    : config.regular_delivery_days_base

      l) Build and return ShippingQuoteOutput

8. EXCEPTION HANDLING / ERROR RESPONSES
─────────────────────────────────────────────────────────────
All errors must return RFC 7807 (API Platform default).

8.1 Custom Exceptions (extend \RuntimeException):
      UnresolvablePostalCodeException   → HTTP 422
      OversizedParcelException          → HTTP 422
      NoTariffFoundException            → HTTP 500

8.2 Exception → HTTP mapping via API Platform's ExceptionToStatusCodeMapping
    in config/packages/api_platform.yaml:
      exception_to_status:
        App\Exception\UnresolvablePostalCodeException: 422
        App\Exception\OversizedParcelException: 422
        App\Exception\NoTariffFoundException: 500

8.3 Validation errors (constraint violations on DTO) are handled
    automatically by API Platform → 422 Unprocessable Entity with
    violations array. No custom code needed.

13. README SECTION (to append to README.md)
─────────────────────────────────────────────────────────────
## Shipping Quote API

### Endpoint Contract

POST /api/shipping/quotes
Content-Type: application/json

Request:
{
  "postalCode": "10000",
  "weight": 2.4,
  "dimensions": { "length": 30, "width": 20, "height": 15 },
  "serviceType": "regular"        // "regular" | "priority"
}

Response 200:
{
  "price": 5.49,
  "currency": "EUR",
  "serviceType": "regular",
  "estimatedDeliveryDays": 3,
  "breakdown": {
    "base": 4.50,
    "weightSurcharge": 0.00,
    "dimensionalSurcharge": 0.00,
    "zoneSurcharge": 0.00,
    "priorityMultiplier": 1.0
  }
}

Error responses: RFC 7807 JSON (API Platform default)
  422 — validation failure, unknown ZIP, oversized parcel
  400 — malformed payload

### Configuration Knobs

Edit config/packages/shipping.yaml:
  volumetric_divisor          default 5000 (use 6000 for some carriers)
  priority_multiplier         default 1.5
  regular_delivery_days_base  default 3
  priority_delivery_days_base default 2
  max_weight.regular          default 30 kg
  max_weight.priority         default 30 kg
  max_dimension.regular       default 150 cm
  max_dimension.priority      default 150 cm

### Zone Definitions

Zones (mainland / island / remote) and their surcharges are stored in
the shipping_zones DB table. Edit via SQL or an admin panel.

### Tariff Tables

Weight brackets per zone are stored in weight_tariffs DB table.
To reseed: php bin/console doctrine:fixtures:load --no-interaction
For custom tariffs, edit src/DataFixtures/WeightTariffFixtures.php and
re-run fixtures (or apply migrations with manual SQL).

### Seeding Postal Codes

A representative subset is seeded via PostalCodeFixtures.
For the full Hrvatska Pošta dataset, run:
  php bin/console app:import-postal-codes path/to/hp_zipcodes.csv
(Console command not yet implemented; add as future enhancement.)
